Stilya 開発ロードマップ（MVP）
フェーズ0：準備・設計（Week 1）
タスク	内容
✅ 要件整理	ペルソナ、ユースケース定義、機能要件の確定
✅ 技術スタック選定	React Native (Expo), Supabase, TypeScript, Tailwind（Expo用UIライブラリ）
✅ デザイン準備	Figmaでワイヤーフレーム・スタイルガイド作成
✅ データ設計	DBスキーマ設計（ER図）、API設計仕様書の完成
✅ アフィリエイト連携	LinkShare / A8.net / 楽天などのAPI調査・認可申請

フェーズ1：MVP機能の実装（Week 2〜4）
タスク	内容
🔧 ユーザー認証	サインアップ／ログイン機能（Supabase Auth）
🔧 商品表示	商品リスト取得API・UIカード表示（スクロール or ページング）
🔧 スワイプ機能	スワイプUIとYes/No記録の保存
🔧 好み学習ロジック	スワイプ結果の簡易集計とレコメンド（タグベース）
🔧 レコメンド画面	パーソナライズ商品提案ページの表示
🔧 アフィリエイトリンク遷移	外部購入ページへ遷移 + クリックログ保存

フェーズ2：テスト・改善（Week 5）
タスク	内容
🧪 ユーザーテスト	クローズドテスト用にTestFlight / Androidビルド配布
🐞 バグ修正	レポートされたUI／ロジック不具合の修正
🧠 ユーザーの声反映	初期ユーザーのフィードバックからUI/UX改善案の抽出
📊 分析	CTR, CVR, スワイプ傾向などの初期データ分析を実施

フェーズ3：限定公開＆プロモーション（Week 6〜）
タスク	内容
📲 アプリ公開	App Store / Google Playのベータ版提出（審査対応）
📢 SNS展開	Instagram / Xで診断型コンテンツ連携と拡散
💬 ユーザー募集	スタイル診断などでのバイラル拡散とフィードバック収集
💰 売上確認	アフィリエイト収益の発生状況確認（収支感覚）

将来的な拡張（Post-MVP）
項目	内容
✅ 画像ベースAI推薦	CLIP・類似画像レコメンド精度の強化
✅ スタイリング記事連携	ブログ・SNS連携によるファッション解説導入
✅ 会員ランク制度	行動量に応じた推薦精度の強化や限定公開枠などの導入
✅ スタイリストとの連携	専門家と提携した診断精度の向上・信頼性向上
