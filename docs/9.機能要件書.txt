機能要件書 – Stilya（MVP）
1. 基本情報
項目	内容
サービス名	Stilya
対象プラットフォーム	iOS / Android（React Native + Expo）
ユーザー	10〜40代のファッションに関心がある男女個人ユーザー
サービス内容	Yes/Noスワイプによる好み学習 → パーソナライズ推薦／購入

2. 機能一覧
ID	機能名	概要
F-01	ユーザー登録／認証	メールアドレスで新規登録・ログイン機能（Supabase Auth）
F-02	スワイプUI	商品カードを左右にスワイプ（Yes/No記録）
F-03	商品表示	商品データ（画像／価格／ブランドなど）のカード表示
F-04	好み記録	スワイプごとにユーザーの嗜好データをDB保存
F-05	商品推薦	ユーザーの嗜好に基づく商品リスト推薦表示
F-06	商品詳細表示	商品カードをタップすると詳細情報と購入ボタン表示
F-07	外部遷移	アフィリエイトリンクから外部ECへ遷移（成果報酬）
F-08	マイページ（簡易）	スワイプ履歴／お気に入り商品表示（初期は履歴のみ）
F-09	商品データ登録バッチ	LinkShare APIから商品情報を定期取得・DB登録
F-10	簡易分析ロジック	タグベースによる推薦ロジック実装（MVP段階）

3. 各機能の詳細要件
F-01：ユーザー登録／認証
項目	内容
入力項目	メールアドレス、パスワード
外部連携	Supabase Auth
バリデーション	メール形式・パスワード6文字以上
エラーハンドリング	ログイン失敗、メール送信失敗時に表示

F-02：スワイプUI
項目	内容
UI形式	Tinder風カードUI
操作	左スワイプ → No、右スワイプ → Yes
状態保存	Supabaseに即時記録（swipesテーブル）
表示制御	スワイプ済商品は再表示しない

F-03：商品表示
| 表示項目 | 商品画像／ブランド／価格／カテゴリタグ |
| デザイン | 1商品1画面でカード形式表示（ネイティブアニメーション対応） |
| 表示数 | 初期ローディング：20件（スクロール対応準備） |

F-04：好み記録
| 保存データ | user_id／product_id／スワイプ結果（yes/no）／タイムスタンプ |
| 保存方法 | Supabase swipes テーブル |
| 利用目的 | 推薦モデル構築用データとして使用 |

F-05：商品推薦
| ロジック（初期） | スワイプ「Yes」が多いタグと一致する商品を抽出 |
| ロジック拡張 | CLIPベース画像特徴類似マッチ（将来対応） |
| 表示形式 | 商品リスト or スワイプ継続型推薦 |

F-06：商品詳細表示
| 表示項目 | 商品画像、ブランド名、価格、タグ、説明文、購入ボタン |
| 遷移方法 | 商品カードタップ時にモーダル or 詳細画面に遷移 |

F-07：外部遷移（購入）
| 遷移元 | 商品詳細画面「購入する」ボタン |
| 遷移先 | アフィリエイトリンク（LinkShare）先の外部ECサイト |
| ログ記録 | 遷移日時、商品ID、userIDを保存可能（任意） |

F-08：マイページ（簡易）
| 表示項目 | スワイプ済商品履歴（Yesのみ）／お気に入り |
| 操作 | 商品詳細画面から「お気に入りに追加」可能（将来） |

F-09：商品データ登録バッチ
| 方法 | Node.jsスクリプトまたはSupabase Edge Functions |
| データ取得元 | LinkShare API（楽天・ブランド横断） |
| 項目 | 商品名／画像URL／価格／カテゴリ／アフィリエイトURL／タグ |
| 保存先 | Supabase products テーブル |

F-10：簡易分析ロジック
| ロジック内容 | スワイプ結果をタグごとにスコア化し、類似スコアの高い商品を表示 |
| タグ例 | メンズ／モード／カジュアル／モノトーン／レトロ など |
| 利用技術 | TypeScriptでロジック実装（MVP）／機械学習は次フェーズ |

4. 非機能要件（重要）
パフォーマンス：初回ローディング1秒以内

セキュリティ：認証トークンはSecureStorageで保存

アクセシビリティ：大きめのボタンUI、ジェスチャー補助あり

マルチデバイス対応：iPhone SE〜Pro Max、Android各種検証
